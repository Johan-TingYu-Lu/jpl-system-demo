# 收費單修復計畫 (ROADMAP)

> 產生時間: 2026-03-29
> 影響範圍: 17 位學生，約 30-40 筆 invoice

---

## 問題根因

1. **之前清理 DB 時，誤刪了正確的 invoice（每位的第 4 或第 5 張）**
2. **Billing engine 從錯誤的 FLAG（最後結束日）開始，自動生成了日期錯位的 invoice**
3. **結果：DB 的計費期間跟 Sheet 完全不同，無法生成 Sheet 要求的新收費單**

## 修復步驟

### Phase 1：清除錯誤資料

**目標：刪除 billing engine 錯誤生成的 invoice 和相關 payment**

1. 找出每位學生「錯位的 invoice」（跟 Sheet 計費期間不匹配的）
2. 刪除這些 invoice 的 payment 紀錄（如有）
3. 刪除這些 invoice 本身
4. 驗證：每位剩下的 invoice 都跟 Sheet 的前幾期吻合（+1 天偏移以內）

**預計刪除：約 25 筆 invoice + 對應 payment**

### Phase 2：修正日期偏移

**目標：將 DB 現有 invoice 的 start_date / end_date 對齊 Sheet（消除 +1 天偏移）**

1. 讀取 Sheet 計費日期表的所有日期
2. 對每位學生的每張 invoice，用 Sheet 日期覆蓋 DB 日期
3. 驗證：DB 日期 = Sheet 日期，零偏移

**預計更新：約 50-60 筆 invoice 的日期**

### Phase 3：重建缺少的 invoice

**目標：用 Sheet 日期重建被誤刪的 invoice**

1. 讀取 Sheet 計費日期表，找出每位缺少的期數（Phase 1 刪除的 + 原本就缺的）
2. 讀取 Sheet 繳費金額表，取得每期金額
3. 讀取 Sheet 繳費日期表，取得繳費狀態
4. 在 DB 建立 invoice：
   - serial_number：按正確序號（補回缺的 04、05 等）
   - start_date / end_date：用 Sheet 日期
   - amount：用 Sheet 金額
   - status：Sheet 有繳費日 → paid，否則 → draft
5. 建立對應 payment 紀錄（如 status=paid）
6. 推送計費日期到 Sheet（pushBillingDates）

**預計新增：約 25-30 筆 invoice**

### Phase 4：修正根因（防止再發生）

**目標：修正 serialToDate 的 +1 天偏移，讓未來生成的 invoice 日期與 Sheet 一致**

1. 檢查 `serialToDate()` 和 `parseUTCDate()` 的時區處理
2. 修正轉換邏輯，確保 Excel serial → Date → DB 存儲 → 讀取全程零偏移
3. 修正 `formatDate()` / `formatDateUTC()` 的使用一致性
4. 單元測試驗證：常見 serial 值轉換後日期正確

### Phase 5：生成新收費單（Sheet 應製單=1 的 16 位）

**目標：為 Sheet 標示「應製單數=1」的學生生成新一期收費單**

1. Phase 1-4 完成後，DB FLAG（最後結束日）已正確
2. Billing engine 應能正確偵測可開單學生
3. 逐一在收費管理頁面確認並生成
4. 生成 PDF 下載
5. pushBillingDates 自動同步到 Sheet

**預計生成：16 筆新 invoice（扣掉 583 林胤呈繳款異常）**

### Phase 6：全面驗證

**目標：確認 DB、Sheet、雲端三方完全一致**

1. 重跑 compare-19-sorted 腳本，確認所有日期差異 = 0
2. 重跑 compare-payment，確認 Sheet→DB 差異歸零
3. 確認 Sheet 學費收支總表「應製單數」全部歸零
4. 確認雲端部署版本與本機一致
5. 線上操作驗證：同步確認頁無異常差異
